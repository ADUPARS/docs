# The Redux API

The Redux API serves as a singular interface into Redux. You can do virtually anything within Redux via the API with simplicity in mind.

API Method Names Have Changed

A number of API methods have changed name in Redux 4.x to match PHP naming conventions. There are still shims for the old names, but they will give you a deprecation notice. For a full list of method changes for the API look below at Deprecated Methods or the Redux 4.x Migration & Compatability.

# History

Working with Redux in the past was much more combersome. Filters, extension loaders, separate arrays, fallback checks. None of these were had in the past. In Redux 3.5+ the API was created to rememdy this. It has futher been enhanced in Redux 4.x+. It is advised to use the new API methods when possible.

# Usage and Examples

Redux can have an unlimited number of sessions running at once. Hence, the opt_name is so crucial. It must be unique, and it is the first variable for the methods within the Redux API.

For safety reasons, and to ensure Redux has loaded properly, it is advised to put a if statement before you use the API in case Redux has not been loaded yet. code.

if ( ! class_exists( 'Redux' ) ) {
    return;
}

# Redux::setArgs()

This method allows you set set global args for your instance of Redux. It's use is required for virtually any Redux instance. An example of how to use it is below.

$opt_name = 'YOUR_OPT_NAME';
$args = Array(
    'opt_name'    => $opt_name,
    'customizer'  => true
);

Redux::setArgs ( $opt_name, $args );

# Redux::getArgs()

This method can take one or two parameters depending on the desired output.

# Fetch All Arguments

The return value is an array of all global arguments for the opt_name instance.

$args = Redux::getArgs( $opt_name );

# Fetch a Single Argument

If a key is provided, and that value is set, the single value will be returned, otherwise null is returned.

echo Redux::getArg( $opt_name, 'opt_name' );

The above value will return a text value that will equal the opt_name string value.

# Redux::setHelpTab()

This method requires two parameters, the opt_name string value, followed by an array value of tab data.

$opt_name = 'YOUR_OPT_NAME';
$tabs = array(
    array(
        'id'      => 'redux-help-tab-1',
        'title'   => 'Theme Information 1',
        'content' => '<br />This is the tab content, HTML is allowed.<br />')
    ),
    array(
        'id'      => 'redux-help-tab-2',
        'title'   => 'Theme Information 2',
        'content' => '<br />This is the tab content, HTML is allowed.<br />')
    )
);
Redux::setHelpTab( $opt_name, $tabs );

# Redux::setHelpSidebar()

This method requires two parameters. First, the $opt_name string value, the second, the $content string value.

$opt_name = 'YOUR_OPT_NAME';
$content = '<br />This is the sidebar content, HTML is allowed.<br />');
Redux::setHelpSidebar( $opt_name, $content );

# Redux::init()

This method forces Redux to load at that instance instead of waiting for the init hook as is default with Redux. This is useful if you want to be able to use your Redux values instantly. Beware, doing so too early can cause issues especially with extensions like metaboxes.

$opt_name = 'YOUR_OPT_NAME';
Redux::init( $opt_name );

# Disable Demo Mode Notice

When the Redux plugin is activated a Activate Demo Mode admin notice is displayed allowing users to try the Redux sample config file. Calling this method will disable that from happening.

Redux::disable_demo();
